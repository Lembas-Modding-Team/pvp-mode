#
# This file lists all gear items that are blocked under certain conditions (alignment, pledging, PvP, PvE, faction).
#
# You can assign a faction an alignment value and a list of items. Players can only use these items if they have at least the specified alignment with the specified faction.
# It's also possible to specify - in addition to the alignment criterion - that the player needs to be pledged to the specified faction.
#
# You can include multiple lines starting with the same faction, but with different alignment values. This way you can specify different conditions for different items.
# Note that if you specify two entries with the same faction, but different alignment values, for the same item, the condition with the lowest alignment value applies.
#
# If a placeholder is specified, the alignment condition has to apply to at least one faction assigned to the placeholder.
#
# A configuration entry consists of one line, which contains three or four columns separated by a semicolon (;).
#
# The first part is the faction name, which is the name which is internally used by the game (see the /alignment command, or use the list on the wiki: https://lotrminecraftmod.fandom.com/wiki/Alignment#List_of_all_Alignment_Factions). You can also use the specifier ALL,
# which means that every player, regardless of his alignment, won't be able to use that item. Note that if you have blocked the usage of an item with ALL, you must make sure
# that there is no other condition which would allow the usage of the item - otherwise the item is usable under that condition.
#
# The second part is the alignment specifier. It's defined with minAlignmentNoPledge/minAlignmentWithPledge. 
# "minAlignmentNoPledge" is the minimum required alignment if the player is not pledged to the specified faction, "minAlignmentWithPledge" is the minimum required alignment if the player is pledged to the specified faction.
# If you only want to specify the alignment value without pledging, just write the number, without anything else. In this case, it is not taken into account whether the player is pledged to the specified faction or not.
# Write "/minAlignmentWithPledge" to only specify the alignment value with pledging. This means that the player has to be pledged to the specified faction.
# If the specified faction is ALL, the alignment specifier will be ignored. (But it still has to be valid!)
#
# The third part is optional - it's the condition, under which the entry will be applied. Valid values are PVP, PVE and ALWAYS. If not specified, ALWAYS is assumed.
# Under this condition the items will be blocked. For example, with PVP as condition the specified items will only be blocked for PvP if the alignment criterion is not fulfilled.
#
# The fourth (or third if no condition is specified) part is a comma-separated list of unlocalized item names (for example, lotr:item.coin), where the specified condition applies.
#
# If this file is deleted, it'll be recreated with default configuration data.
# On startup you'll see in the logs if the PvP Mode Mod found errors in this file, so that you can fix them.
#
# Some words to the file format: 
# Comments cover a whole line, which must start with a #. The PvP Mode Mod will ignore them.
# Blank lines and spaces will be ignored too - you can put as much spaces between the configuration entries as you want.
#
